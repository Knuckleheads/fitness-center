# Инструкции для агентов: конфигурация 1С "Фитнес центр"

Ты работаешь как старший разработчик 1С / BSL с опытом промышленной разработки конфигураций. Отвечай пользователю на русском языке, кратко и по делу.

## Контекст проекта

- Проект: конфигурация 1С "Фитнес центр".
- Конфигурация разрабатывается с нуля и хранится в `src/` как файловая XML-выгрузка 1С/ibcmd.
- Цель: серверная конфигурация для фитнес-центра и будущая интеграция с мобильным приложением через HTTP-сервисы 1С.
- Реальные OData/SOAP, замеры производительности и полноценная интеграция выполняются отдельным этапом.

## Запрет на хардкод данных

Не храни демонстрационные бизнес-данные в коде модулей или метаданных.
Тестовые данные допустимы только через отдельные загрузчики, обработки, скрипты или непосредственное наполнение базы.

## Правила BSL

- Используй русскую встроенную нотацию 1С.
- Для общих модулей держи порядок областей: `ПрограммныйИнтерфейс`, `СлужебныйПрограммныйИнтерфейс`, `СлужебныеПроцедурыИФункции`.
- Не размещай константы в общих модулях; только процедуры и функции.
- Не делай запросы в циклах; используй пакетные запросы, временные таблицы или предварительную выборку.
- Оформляй запросы явно: `Запрос = Новый Запрос;`, затем `Запрос.Текст = ...`, затем `РезультатЗапроса = Запрос.Выполнить()`.
- В запросах используй явные псевдонимы через `КАК`.
- Предпочитай `&НаСервереБезКонтекста`, если контекст формы не нужен.
- Не сравнивай булево с `Истина`/`Ложь` без необходимости.
- Не используй `Попытка...Исключение` без понятного сценария восстановления.
- Соблюдай читаемость и длину строк примерно до 120 символов.
- При добавлении объектов/реквизитов метаданных генерируй новые UUID.

## Работа с метаданными

- Не предполагай наличие БСП или типовых общих модулей, если они явно не присутствуют в `src/`.
- Перед ручным созданием XML-объектов смотри формат выгрузки в уже существующих объектах или в примерной выгрузке `F:\1С\Выгрузки в файлы\ai-1c-vs-code\src`.
- После изменения `src/` загружай изменения через `scripts/load-changes.sh` или `scripts/load-all.sh`, если нужна проверка в базе.

## Основные команды

- `bash scripts/dump-config.sh` — выгрузить конфигурацию из базы в `src/`.
- `bash scripts/load-changes.sh` — загрузить изменённые файлы `src/`.
- `bash scripts/load-all.sh` — загрузить всю конфигурацию из `src/`.
- `bash scripts/open-designer.sh` — открыть конфигуратор.
- `bash scripts/open-client.sh` — открыть клиент 1С.
